# Vision Document - FileSystem Emulator

## Мета Проекту

Розробити освітній мініпроєкт, який демонструє практичне застосування design патернів (Composite, Command, Proxy) та SOLID принципів на прикладі емулятора файлової системи.

## Бачення

Проект призначений для:

1. **Навчання Design Патернів**
   - Composite для древовидної структури даних
   - Command для операцій з можливістю Undo
   - Proxy для контролю доступу та логування

2. **Застосування SOLID Принципів**
   - Single Responsibility Principle
   - Open/Closed Principle
   - Liskov Substitution Principle
   - Interface Segregation Principle
   - Dependency Inversion Principle

3. **Демонстрація Сучасних Практик C#**
   - .NET 9 та C# 13 синтаксис
   - Generic колекції та типізація
   - LINQ для запитів
   - JSON серіалізація
   - xUnit тестування

## Цільова Аудиторія

- Студенти, які вивчають design патерни
- Розробники, які хочуть вдосконалити навички архітектури
- Інструктори, які потребують навчального матеріалу

## Технічні Рішення

### Чому C# 13 та .NET 9?
- Сучасний язык з потужною типізацією
- Вбудована підтримка generic типів
- Відмінна екосистема для навчання
- Крос-платформне рішення

### Чому Composite для файлової системи?
- Природна ієрархія файлів і папок
- Рекурсивна обробка структури
- Єдиний інтерфейс для різних типів елементів
- Розширюваність без модифікації

### Чому Command Pattern для операцій?
- Можливість скасування операцій
- Логування всіх дій
- Командна черга для послідовного виконання
- Розділення запиту від виконавця

### Чому Proxy для контролю доступу?
- Прозора перевірка прав
- Логування операцій
- Можливість кешування прав
- Динамічна конфігурація доступу
