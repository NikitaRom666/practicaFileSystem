# Завдання 6: SOLID принципи та патерни

## Хід роботи

### Реалізовані Design Патерни

#### 1. COMPOSITE (Композит)
Паттерн реалізовано через `DirectoryItem` та `FileSystemItem`:
- Директорія містить колекцію `List<FileSystemItem>`
- Методи `GetSize()`, `Print()`, `Search()` викликаються рекурсивно через дітей
- Файл та каталог взаємозамінні при роботі через базовий тип
- Це дозволяє легко складати складні структури без спеціального код для різних типів елементів

#### 2. COMMAND (Команда з Undo)
Паттерн реалізовано через `ICommand` інтерфейс:
- `CopyCommand`, `MoveCommand`, `DeleteCommand` інкапсулюють операції
- Кожна команда має `Execute()` та `Undo()`
- `CommandHistory` зберігає стек команд і дозволяє скасовувати операції (максимум 20)
- Команди логуються: `[CMD] Executed: Copy file.txt to Backup`

#### 3. PROXY (Проксі)
Паттерн реалізовано через `FileSystemProxy`:
- Обгортає доступ до файлової системи та перевіряє права перед операціями
- `CheckAccess()` кидає `AccessDeniedException` якщо прав недостатньо
- Адміністратор завжди проходить перевірку без додаткових перевірок
- Логування: `[LOG] alice читає document.txt`

### SOLID принципи

#### Single Responsibility Principle (SRP)
Кожен клас має одну чітко визначену відповідальність:

- **FileItem** — представляє файл з вмістом
- **DirectoryItem** — управляє колекцією вложених елементів
- **DiskVolume** — управляє диском (розмір, root каталог)
- **FileSystemUser** — представляє користувача з роллю
- **FileSystemProxy** — тільки перевірка прав доступу
- **SerializationService** — тільки серіалізація/десеріалізація
- **FileSystemQueryService** — тільки LINQ запити
- **CommandHistory** — тільки управління історією команд

Ця розподіленість робить код легко розширяти та змінювати.

#### Open/Closed Principle (OCP)
Код відкритий для розширення, але закритий для модифікації:

- Нові команди додаються через реалізацію `ICommand`, без змін у `CommandHistory`
- Нові типи файловий систем додаються через наслідування `FileSystemItem`
- Нові сервіси можна додати без змін існуючих компонентів
- Розширення через inheritance: якщо потрібна нова операція, просто додаємо новий Command клас

#### Liskov Substitution Principle (LSP)
Подекласи можуть замінити своїх батьків без порушення поведінки:

- `FileItem` та `DirectoryItem` можна використовувати як `FileSystemItem`
- Код працює з `FileSystemItem[]` поліморфно, не знаючи конкретного типу елемента
- Кожне розширення `FileSystemItem` коректно реалізує контракт базового класу
- `Print()` та `GetSize()` працюють однаково незалежно від типу елемента

#### Interface Segregation Principle (ISP)
Клієнти залежать тільки від того, що їм потрібно:

- `IFileSystemItem` — мінімальний набір операцій для всіх елементів
- `ISearchable` — тільки для `DirectoryItem` (не впихаємо в `FileItem`)
- `IPrintable` — для всіх що можуть виводитися
- `ICommand` — для операцій
- `IFileSystemProxy` — для Proxy операцій
- Клієнти не залежать від методів, які вони не використовують

#### Dependency Inversion Principle (DIP)
Модулі залежать від абстракцій, а не від конкретних реалізацій:

- `FileSystemProxy` залежить від `ICommand`, а не від конкретних команд
- `CommandHistory` приймається як параметр до `Proxy`, а не створюється всередині
- `FileSystemQueryService` використовує IEnumerable для повернення результатів
- Заміна реалізацій можлива без змін клієнтського коду

### Обробка виключень

Спеціальні винятки забезпечують точний контроль помилок:

- `AccessDeniedException` — при відсутності прав
- `ItemNotFoundException` — коли елемент не існує
- `ItemAlreadyExistsException` — дублювання імена
- `DiskFullException` — переповнення диска
- `InvalidFileSystemOperationException` — неправильна операція

Це дозволяє точно обробляти різні помилкові ситуації.

### Демонстрація у консолі

Програма демонструє:
1. Створення структури файлів та папок
2. Вивід дерева структури з іконками
3. Статистику дискового простору
4. Copy операцію з командою
5. Скасування (Undo)
6. Пошук файлів за шаблоном
7. Валідацію структури

Вихідні логи показують: `[CMD]`, `[LOG]`, `[UNDO]` тощо для отримання інформації про операції.
