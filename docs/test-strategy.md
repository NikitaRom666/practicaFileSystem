# Стратегія тестування

## Підхід

Unit-тестування окремих модулів в ізоляції. Структура тестів: AAA (Arrange, Act, Assert).

## Фреймворки

- xUnit — основний фреймворк тестування
- Moq — мокування залежностей

## Що тестується

- Command pattern: виконання та скасування операцій
- CommandHistory: додавання, скасування, обмеження розміру стеку
- FileSystemItem та його похідні: розмір, пошук, клонування
- FileSystemProxy: перевірка прав, виключення при порушенні

## Критерії прийняття

- Всі тести проходять (0 failed)
- Кожна команда має тест на Execute та Undo
- Proxy має тест на дозволену та заборонену операцію
